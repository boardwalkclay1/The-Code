The Code — Full Definitions & How They Apply

This page breaks down the core abbreviations and concepts you’ll see everywhere in coding. Once you understand these, you’ll be able to read code, fix errors, and guide AI with confidence.

Even though AI can write most of your code today, you still need to understand the structure. That’s how you spot mistakes, understand what’s happening, and build anything you want.

There has never been a better time to get into coding.

Core Abbreviations & Concepts

HTML — HyperText Markup Language

The structure of every webpage.

How it applies: HTML is the skeleton of your site.

CSS — Cascading Style Sheets

The design layer.

How it applies: CSS makes your project look professional.

JS — JavaScript

The behavior layer.

How it applies: JS makes your site interactive.

DOM — Document Object Model

The browser’s internal map of your webpage.

How it applies: JS uses the DOM to change elements.

API — Application Programming Interface

How apps talk to each other.

How it applies: APIs let your app pull real data.

URL — Uniform Resource Locator

The address of a webpage.

How it applies: Every GitHub Pages file has a URL.

HTTP / HTTPS

The protocol your browser uses.

How it applies: HTTPS is required for modern features.

IDE — Integrated Development Environment

Your coding editor.

How it applies: VS Code is where you build everything.

CLI — Command Line Interface

A text‑based interface for commands.

How it applies: You’ll use it to push code to GitHub.

Repo — Repository

A folder of code stored on GitHub.

How it applies: Every project lives in a repo.

Commit

A saved change in your code.

How it applies: Lets you undo mistakes instantly.

Push

Sending your code to GitHub.

How it applies: Updates your project online.

Deploy

Publishing your project.

How it applies: GitHub Pages deploys sites for free.
